Memory Leak Simulator - Stack & Heap

STACK
Lo stack è una regione di memoria che alloca e dealloca automaticamente le variabili locali quando le funzioni vengono chiamate e restituite.
HEAP
L'heap è una regione di memoria per allocazioni dinamiche. La memoria deve essere gestita manualmente (in linguaggi come C) o dal garbage collector.

Differenze tra Stack e Heap

Stack: Memoria automatica e veloce, ma limitata in dimensione. Usata per variabili locali e chiamate a funzione. La gestione è automatica (LIFO - Last In First Out).

Heap: Memoria dinamica più grande ma più lenta da gestire. È qui che si verificano i memory leak quando la memoria allocata non viene liberata.

Memory Leak con puntatori

Quando un puntatore nello stack (variabile locale) viene perso (perché la funzione termina o perché il puntatore viene sovrascritto), ma la memoria nell'heap a cui puntava non viene liberata, si crea un memory leak. Non c'è più modo di accedere a quella memoria, ma rimane allocata.

  2. Clicca sullo stack per creare un puntatore a quella memoria
  3. Premi "Libera Stack" per simulare l'uscita dallo scope della funzione
  4. Nota che la memoria nell'heap rimane allocata (memory leak)
